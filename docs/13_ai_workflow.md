# AI 协作工作流

本页记录与 AI 助手协作时的日志、备份和交接纪律,便于长期流程复现与接手。

## 单一主日志

一个项目只维护一份 Markdown 主日志,所有决策/试错/结果按时间追加。每个工作段:
目标 → 假设 → 命令(带目录与参数)→ 输出摘要(精确数字)→ 结论/下一步。
失败也记:失败点、完整报错、试过的修复。AI 上下文会被压缩——日志就是外置记忆。

## 备份纪律

- 每轮完整流程出结果 → `backups/<tag>_<日期>/` 存报告+脚本快照;
- RTL 每版单独存(v5_2.v ... v5_11.v),当前活跃版不带版本号;
- 里程碑打 tar 包 + sha256,服务器/本地双备份;
- 命名带语义:`*_FULL_PASS_*`、`*_FINAL_*` 一眼可辨可回滚点。

## 远端执行

用 tools/remote/dprun.sh(ssh key 免密):push 代码 → exec 跑阶段 → pull 报告。
EDA 大产物不回传,只回传判定行所需的报告;AI 读判定行决定下一步。

## 交接清单(换 AI / 换人继续时)

主日志、当前 RTL+tb、最新一轮报告目录、最近完整备份(含校验)、env/config 脚本。
接手第一件事:跑 collect_reports.sh 核实"声称的状态"与"实际文件"一致。

## skills 的角色

- `digitalpilot`:流程驱动(每阶段命令+判定+决策树);
- `lib-detective`:缺失模型自助定位(报错→形态→路径→固化进脚本)。
装到 `~/.claude/skills/` 后,AI 遇到对应触发场景会自动应用这些方法,
不再依赖人类口口相传。
