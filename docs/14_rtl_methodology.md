# RTL 架构与验证方法学(题目无关)

本仓库不含任何题目的 RTL 答案;本文是从原型项目蒸馏出的**通用设计方法**,
配合流程脚本,使你(或 AI 助手)能对任意新题目独立完成设计与交付。

## 第一步:把题面读成"硬规格"(最容易翻车的环节)

逐条抽取并写进设计笔记,**每条都要在 testbench 里对应一个检查**:

1. **公式语义**:注意定点缩放。形如 `y = f(...)/2^N` 且"y 为有符号小数"时,
   硬件输出往往是 `Y = y·2^N` 的整数(Q 格式)——先把"规格式"换算成"硬件式"
   再设计,报告里两式都给并写清推导;
2. **串行协议**:位序(MSB-first 还是 LSB?题面会明说,做反判零分)、
   多少拍一组、写满后**何时生效**(常见"下一拍生效")、复位打断后从哪一位重来;
3. **latency 定义**:第 t 拍输入对应第 t+L 拍输出,L 固定;参数更新边界也要满足同一 L;
4. **饱和/舍入**:正负饱和值、舍入规则(round-to-nearest?向零?),输出位宽;
5. **覆盖清单**:题面"功能验证要求"逐条编号,直接变成 tb 的用例清单。

## 架构套路(GF180 @ 200 MHz 的经验值)

- **每级组合逻辑预算**:5 ns 周期 - 1 ns uncertainty - 触发器开销 ≈ 3.6 ns(WST),
  对应 **Levels of Logic ≈ 20** 上限。乘法/除法必须切流水;
- **乘法**:12×12 拆成 高/低位部分积两级(6×12 + 6×12,再一级合并),
  或更宽的拆 8×N 三段;DC 综合的 `qor.rpt` Levels 超 20 就再切一刀;
- **除法**:radix-4 恢复余数除法器,每级出 2 位商(预计算 1/2/3 倍除数,
  三路并行减法选最大不负者),12 位商 = 6 级流水;末级余数做舍入;
- **周期函数(cos/sin)**:四分之一波 LUT + 象限折叠(索引/符号两个小函数),
  LUT 用脚本生成(见 `scripts/utils/gen_lut.py`),**逐项 = round(f·2^N),0 偏差**;
- **位宽推导**:对每个中间量算最大值并留出表格(报告必考)。例:两个 12bit
  无符号乘积 < 2^24;再乘 13bit 幅值 < 2^37——切勿拍脑袋,差 1 bit 就溢出;
- **reset 策略**:输出寄存器/输入采样/串行参数装载用 RAW reset(保证 reset 期间
  输出为 0、与 tb 逐拍对齐);**深流水线各级可用打一拍的 reset_reg** 门控——
  把"输入→全设计广播"的长组合路径变成 reg-to-reg,消 WST setup 违例,
  且功能等价(reset 期间上游本来就是 0)。这是原型实测省下 ~2ns 的关键一招;
- **串行参数装载**与计算解耦:装载是后台移位 + 写满锁存 + 快照寄存器,
  计算每拍照常用"当前生效值",吞吐恒为 1/拍——不要把两者搅在一个状态机里。

## 验证纪律(黄金模型三原则)

1. **独立实现**:tb 的 model 用实数运算($cos 等)+ 与规格相同的舍入/饱和,
   绝不照抄 RTL 结构——抄了就失去比对意义;
2. **逐拍比对 + LATENCY 移位队列**:`y === exp_pipe[LATENCY-1]`,错即打印
   cycle/期望/实得;`checks` 总数是三仿一致性的硬指标;
3. **协议状态同构**:tb 里的参数装载状态机与 RTL 的递推**逐拍同步**
   (用相同的"写满下一拍生效"语义),否则边界周期必然误报。

定向用例包成宏(每组前 reset、打印 `CASE_n_START/OUTPUT`),答辩时波形好定位;
随机 ≥3000 组;骨架见 `scripts/0_sim_rtl/tb_template.v`。

## 迭代节奏(从 RTL 到收敛)

```
RTL 改动 → make run_vcs(分钟级,先功能闭环)
        → DC(看 qor Levels 与 slack;不过 → 回去切流水,别恋战工具选项)
        → Innovus 批处理(postRoute 0 violating?)
        → 三角签核(WST setup / BST hold 各自卡点,BST hold 用 ECO 收)
```
**优先改架构,其次改约束策略,最后才是工具参数**;时钟周期只许收紧不许放松。
每轮在主日志记录:版本号、改了什么、各阶段数字(docs/13 的纪律)。
